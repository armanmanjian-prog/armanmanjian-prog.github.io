Previous Session Code (Reference)

Study this carefully. You will reuse its core idea in today’s work. 

#include <Servo.h>

Servo servo1;
Servo servo2;

int currentAngle = 90;

void setup() {
  Serial.begin(9600);

  servo1.attach(A1);
  servo2.attach(A2);

  servo1.write(90);
  servo2.write(90);

  Serial.println("=== SMOOTH ARM CONTROL (FOR LOOP) ===");
  Serial.println("Send angle (0–180)");
}

void loop() {
  if (Serial.available() > 0) {

    int targetAngle = Serial.readStringUntil('\n').toInt();
    targetAngle = constrain(targetAngle, 0, 180);

    moveSmooth(targetAngle);

    Serial.print("Moved to: ");
    Serial.println(targetAngle);
  }
}

void moveSmooth(int targetAngle) {

  if (targetAngle > currentAngle) {
    for (int angle = currentAngle; angle <= targetAngle; angle++) {

      servo2.write(angle);
      servo1.write(180 - angle);

      delay(15);
    }
  }

  else {
    for (int angle = currentAngle; angle >= targetAngle; angle--) {

      servo2.write(angle);
      servo1.write(180 - angle);

      delay(15);
    }
  }

  currentAngle = targetAngle;
}

1. Step One: Independent Joint Control

Your first goal is to control each joint independently, one at a time.

What you are building

  • The user enters an angle
  • Only one joint moves
  • The system then asks for the next joint
Think before coding
  • How can the program remember which joint is active?
  • How can you avoid repeating code for every servo?
Hint

Instead of making many separate variables, think about storing information in lists:

  • A list of joint names
  • A list of joint angles

Starter Code for Base and Lift2

Use this as a starting pattern. It shows one way to organize the logic for two joints first. 

#include <Servo.h>

Servo baseServo;
Servo lift2Servo;

// Store the angles for each joint
int angles[2];

// Store the names of each joint
String jointNames[2] = {
  "Base",
  "Lift2"
};

// Keeps track of which joint we are controlling now
int currentJoint = 0;

void setup() {
  Serial.begin(9600);

  baseServo.attach(A0);
  lift2Servo.attach(A1);

  Serial.println("Independent Joint Control");
  askNextJoint();
}

void loop() {
  if (Serial.available() > 0) {
    String input = Serial.readStringUntil('\n');
    input.trim();

    int angle = constrain(input.toInt(), 0, 180);

    // Store the angle in the array
    angles[currentJoint] = angle;

    // Move only the current joint
    moveJoint(currentJoint, angle);

    // Go to the next joint
    currentJoint++;

    // If there are still joints left, ask for the next one
    if (currentJoint < 2) {
      askNextJoint();
    } 
    else {
      Serial.println("All joints entered.");
      currentJoint = 0;
      Serial.println();
      askNextJoint();
    }
  }
}

void askNextJoint() {
  Serial.print("Enter angle for ");
  Serial.print(jointNames[currentJoint]);
  Serial.println(":");
}

void moveJoint(int jointIndex, int angle) {
  if (jointIndex == 0) {
    baseServo.write(angle);
  }
  else if (jointIndex == 1) {
    lift2Servo.write(angle);
  }

  Serial.print(jointNames[jointIndex]);
  Serial.print(" moved to ");
  Serial.println(angle);
}

What this teaches them

This part introduces three important ideas:

1. How the program remembers which joint is active

int currentJoint = 0;

This variable acts like a tracker for the current step in the process. At the beginning, the program starts with joint 0, which is the Base. After that, it moves to joint 1, which is Lift2.

  • 0 means Base
  • 1 means Lift2

2. How names are stored

String jointNames[2] = {
  "Base",
  "Lift2"
};

This allows the program to print the correct joint name automatically. Instead of writing separate Serial messages for every joint, the program can look up the name in the list.

3. How angles are stored

int angles[2];

This creates a place to store the entered angle for each joint. Each index in the array matches one joint.

  • angles[0] stores the Base angle
  • angles[1] stores the Lift2 angle

What students should notice

As they study this code, ask them to identify:

  • Why does currentJoint increase after each input?
  • Why does the program reset currentJoint back to 0?
  • How does moveJoint() know which servo to move?

Follow-up challenges

After students understand this structure, they should extend it on their own.

Challenge 1

Add Lift3 by themselves.

Challenge 2

Change Lift2 so it becomes a mirrored dual-servo joint.

Challenge 3

Make the system print all saved angles after both joints are entered.

2. Observing the Limitation

Once your independent joint control works, something important becomes obvious: the servos move correctly, but the motion is instant.

Reflect
  • Does this look like a real robotic arm?
  • What problems can sudden motion cause?

You should notice that the arm moves in a way that feels abrupt and mechanically rough.

3. The Core Question

Now ask yourself the most important question in this lesson:

How can we move from one angle to another gradually instead of instantly?

Connect to the previous session

  • You already solved this once
  • You used a loop
  • You tracked the current angle

4. Designing Smooth Motion

The goal is to replace direct angle writing with progressive movement.

Think
  • What is the starting angle?
  • What is the target angle?
  • How can the servo travel from one to the other step by step?
Hint

A loop can move through all angle values between the current position and the target position.

Key challenge
  • What if the target is smaller than the current angle?
  • Should the loop always increase?

5. Apply Smooth Motion to One Joint

Now modify your system so one joint moves smoothly instead of instantly.

Think
  • Where should the smooth logic go?
  • Should it stay inside the main loop, or belong in a separate function?
Hint

Focus on the function that moves one joint. That is the best place to begin improving the motion.

6. Expand to All Joints

Once one joint moves smoothly, apply the same idea to every joint in the arm.

Think
  • Can one movement function be reused for all joints?
  • What information does that function need?
Hint
  • Which servo is being controlled?
  • What is the current angle?
  • What is the target angle?

7. Deeper Problem: Sequential Motion

Even after adding smooth motion, there is still another limitation: the joints may move one after another instead of behaving as a coordinated robotic arm.

Reflect
  • Do real robotic arms move only one joint at a time?
  • What would coordinated motion look like?

8. Coordinated Motion

The next challenge is to move all joints together, smoothly.

Think differently
  • Instead of finishing one joint completely, what if all joints move a little during each step?
  • How would that change the movement of the arm?
Hint

Use a loop that updates every joint during each pass, rather than completing one joint before starting another.

9. Multi-Joint Control System

To coordinate motion, you now need a way to store all current angles and all target angles.

Think
  • How can all current joint positions be stored together?
  • How can all target positions be stored together?
  • How can one loop update the entire arm?

10. Defining Positions

In robotics, a single arm position is not one angle. It is a complete set of angles for all joints.

Important idea

A robotic arm position is a collection of values that describe the whole arm at once.

Think
  • How can a full arm position be stored?
  • How can positions be reused later?

11. Final Challenge: Pick and Place

Your final goal is to create a system that:

  • Moves to a pickup position
  • Closes the gripper
  • Moves to another position
  • Opens the gripper
Important rule
  • No sudden jumps
  • All movement should be smooth
  • All joints should work together
Think before coding
  • How can the arm move to a full position smoothly?
  • How should the gripper be controlled?
  • What sequence of actions is needed to pick up and place a box?

12. Building Functional Behavior

By the end of this challenge, your program should naturally grow into a set of useful functions.

Possible function ideas

  • Move to a position smoothly
  • Open the gripper
  • Close the gripper
  • Pick an object
  • Place an object
Hint

Each function should focus on one responsibility. This keeps your program organized and easier to improve.

13. System Thinking

At this point, you are no longer just moving servos. You are building a robotic system that:

  • Takes input
  • Controls multiple components
  • Coordinates motion
  • Performs a task
Final Reflection
  • What is the difference between moving one motor and controlling a robot?
  • Why is smooth motion important in robotics?
  • How could sensors improve this system?

Extension Challenge

Try combining everything you learned into one complete system:

  • Step-by-step user input
  • Independent joint control
  • Smooth movement
  • Coordinated motion
  • Automated pick-and-place behavior
Final Thought

If your robotic arm can move smoothly, coordinate all joints, grab a box, and place it somewhere else, then you have built the foundation of a real robotic control system.
